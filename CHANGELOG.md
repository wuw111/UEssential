# UEssential v1.1.2 更新日志 (Changelog)

**发布日期：** 2026/04/14
**当前版本：** v1.1.2
**开发版本：** v0.8.3
**适配环境：** LeviLamina / LLSE 生存服务器

---

## ✨ 新增功能

### 1. 封禁身份关联补全
封禁系统现已与玩家数据库（PlayerDatabase）深度联动。
- **关联查禁：** 在封禁玩家时，系统会自动调取数据库中记录的所有历史曾用名、关联 IP 及设备 ClientID，实现“一人违规，全身份关联封禁”，有效防止换号绕过。

### 2. 管理员物品克隆 (SuperAdmin)
在 `/playermanage` 的物品 NBT 审查界面中，为超级管理员新增了“物品复制”功能。
- **一键克隆：** 允许超级管理员将正在审查的物品（含完整 NBT 数据）直接复制一份到自己的背包中，便于违规物品取证或损失补偿。

---

## 🛠️ 重要调整与优化

### 1. 模块化审计日志系统
重构了安全模块的日志输出逻辑，提供更透明的拦截反馈。
- **全流程记录：** 反作弊验证结果现已细化并同步至控制台与日志文件。
- **步骤追踪：** 明确区分“本地黑名单检查”与“各云端接口检查”的开始、通过、不通过及接口异常状态，方便排查拦截原因。

---

## 🐞 修复与优化
- **权限校验：** 严格限制物品克隆功能仅对 `config.json` 中配置的 `superAdmins` 开放。
- **容错处理：** 增加了对物品克隆时背包容量的检测，防止因背包已满导致的物品丢失或脚本异常。
- **日志一致性：** 统一了本地封禁与云端封禁在控制台中的输出格式。

---

## ⚠️ 提醒
- **数据迁移：** 本版本无需修改数据库结构，平滑升级。
- **配置建议：** 建议开启 `playerDatabase` 功能以获得最佳的封禁关联效果。

# UEssential v1.1.1 更新日志 (Changelog)

**发布日期：** 2026/04/11
**当前版本：** v1.1.1
**适配环境：** LeviLamina / LLSE 生存服务器

---

## ✨ 新增功能

### 1. 背包物品 NBT 检查 
为了协助管理员进行更精准的违规审查，我们在 `/playermanage` 的玩家状态菜单中引入了 **NBT 实时检查** 功能。
- **实时透视：** 在“查看状态信息”界面点击任何一件背包物品，系统将实时从目标玩家容器中重新获取该物品。
- **格式化展示：** 以易读的层级化文本形式展示该物品完整的 NbtCompound 结构，包含隐藏属性、附魔数据、自定义标签等关键信息。
- **辅助定损：** 无论是排查刷物还是确认丢失物品，均可实现“即点即看”，无需再导出繁琐的数据库文件。

### 2. 跨插件开放 API 支持
UEssential 现已成为服务器的数据中台，通过标准的 `ll.export` 接口为其他插件提供底层数据支撑：
- **TPS 监控：** 提供 `getTps1Min` (一分钟平均) 与 `getTpsInstant` (秒级即时) 接口，让其他性能监控或自动清理插件共享高精度测量结果。
- **时长同步：** 提供 `getOnlineTime` 接口，方便勋章插件或等级插件调用 UEssential 统计的精准在线时长。
- **安全检查：** 提供 `isBanned` 接口，允许其他插件在处理高敏感操作前，快速检查玩家是否处于本地黑名单中。

---

## 🛠️ 重要调整与优化

### 1. 全局 IP 归一化
修复了 Bedrock 版网络协议中端口号变动对数据记录的干扰。
- **标准记录：** 所有涉及 IP 记录的模块（玩家数据库、CSV 审计日志、登录日志）现已统一剥离端口号。
- **查禁增强：** 封禁系统在进行 IP 匹配时不再受随机端口影响，有效防止了通过重连更换端口来规避 IP 封禁的行为。

### 2. 在线时长统计逻辑重构
我们抛弃了传统的“定额累加”模式，引入了基于物理时间差的动态统计算法。
- **精准增量：** 当 TPS 测量功能开启时，系统每 1200 Tick 通过计算**物理时间戳增量**来统计时长。这意味着无论服务器是否因卡顿导致 Tick 变慢，记录的游玩时间都将与现实时间完美同步。
- **平滑降级：** 若 TPS 功能被禁用，系统将自动切换为每 1200 Tick 增加 1.05 分钟的估算模式，确保数据不中断。

---

## 🔗 开发者 API 快速参考

如果您是插件开发者，现在可以按以下方式调用 UEssential 的数据：

const getTps = ll.import("UEssential", "getTpsInstant");
let currentTps = getTps(); // 返回 Float 或 null

- **数据一致性：** 修复了在极端网络波动下，玩家管理界面显示物品信息可能产生延迟的同步问题。
- **存储优化：** `playerdatabase.json` 内部结构优化，减少了冗余端口字符串占用的存储空间。

# UEssential v1.1.0 更新日志 (Changelog)

**发布日期：** 2026/04/08
**当前版本：** v1.1.0
**适配环境：** LeviLamina /LLSE 生存服务器

---

## ✨ 新增功能

### 1. 玩家在线管理系统 (Player Management)
引入了全新的可视化管理工具 `/playermanage`，专为服主和管理员设计。
- **旁观模式 (Spectate)：** 一键切换旁观并传送到玩家处。
- **替身操作 (As...):** 支持以目标玩家身份进行发言 (`talkas`) 或执行命令 (`cmdas`)，用于协助解决问题或合规性测试。
- **实时状态追踪 (Status)：** 
    - 实时查看玩家的基本信息（XUID、IP、设备ID、操作系统等）。
    - **背包透视：** 在 GUI 界面直接查看在线玩家背包内的所有物品及数量。
- **惩罚工具包：** 集成封禁 (Ban)、踢出 (Kick) 及客户端崩溃 (Crash) 功能，极大简化了处理违规玩家的操作流程。

### 2. 玩家行为数据库 (Player Database)
建立了两套本地数据库（`playerdatabase.json` 与 `regplayer.json`），实现玩家信息的全生命周期记录。
- **注册统计：** 自动记录玩家首次进入服务器的时间（精确到秒）。
- **更名记录 (History Name)：** 自动追踪并记录玩家的历史曾用名。
- **精准在线时长：** 
    - **TPS 补偿计算：** 在线时长不再是简单的现实时间累计。系统会根据服务器实时 TPS 动态调整增长速率，在卡顿时自动放缓统计，确保存档数据更加真实。
- **数据库查询：** 使用 `/playerdatabase` 即可查看历史总注册量及近 7 日内新入坑玩家的详细列表。
